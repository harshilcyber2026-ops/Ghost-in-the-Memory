🧠 Memory Forensics · Challenge #07

Ghost in the Memory

A Windows memory dump was captured on a compromised machine. The attacker stored the flag in a running process. Dig it out.

Description

Our incident response team captured a memory dump from a compromised Windows workstation. Initial triage shows notepad.exe was running with unusual heap allocations — the attacker stored sensitive exfiltration data inside its process memory.

The dump contains multiple memory regions: image pages for system DLLs, and a suspicious notepad.exe private heap allocation. One heap buffer contains an active exfiltration payload the attacker was preparing to send.

Analyse the dump. Find the suspicious process. Extract the flag from its heap. Multiple methods work — Volatility, Python regex, strings, or raw hex seek.

Analysis Methods

Method 1 — strings (fastest, no tools needed)
$ strings memory.dmp | grep "FlagVault{" FlagVault{m3m0ry_h4s_n0_s3cr3ts} $ strings memory.dmp | grep -B2 -A4 "ATTACKER_BUFFER_START" [ATTACKER_BUFFER_START] collecting_target: COMPROMISED-PC\attacker timestamp: 2024-11-22T03:14:07Z secret_key: FlagVault{m3m0ry_h4s_n0_s3cr3ts} [ATTACKER_BUFFER_END]
Method 2 — Python (cross-platform, no dependencies)
#!/usr/bin/env python3 — save as extract.py import re with open("memory.dmp", "rb") as f: data = f.read() match = re.search(rb'FlagVault\{[^}]+\}', data) if match: print(match.group(0).decode()) $ python3 extract.py FlagVault{m3m0ry_h4s_n0_s3cr3ts}
Method 3 — Volatility 3
$ vol -f memory.dmp windows.pslist PID PPID ImageFileName Offset 6732 4384 notepad.exe 0x1BCA2000000 ← suspicious $ vol -f memory.dmp windows.memmap --pid 6732 --dump Writing to pid.6732.dmp ... $ strings pid.6732.dmp | grep "FlagVault{" FlagVault{m3m0ry_h4s_n0_s3cr3ts}
Method 4 — Direct byte offset (advanced)
$ python3 -c "d=open('memory.dmp','rb').read(); i=d.find(b'FlagVault{'); print(hex(i), d[i:i+32].decode())" 0xc0a05 FlagVault{m3m0ry_h4s_n0_s3cr3ts} # Or with dd (Linux): $ dd if=memory.dmp bs=1 skip=788997 count=32 2>/dev/null FlagVault{m3m0ry_h4s_n0_s3cr3ts}
📌
About the file size: This is a crafted mini memory dump (~1.1 MB) built for this CTF. Real Windows dumps are typically 2–16 GB. All analysis techniques shown work identically on both. The dump has a valid MDMP header and genuine memory region structures — Volatility can parse the header, and strings/Python will find the flag in the heap region at offset 0xC0A05.
